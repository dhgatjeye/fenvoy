# Security Policy

## Disclaimer

**fenvoy has not been independently audited for security.** No formal third-party review has been conducted. Use at your own risk - see the [README](README.md) for full disclaimer.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately by emailing:
**stackfenvoy@mailfence.com**

Include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Disclosure Policy

We follow a **coordinated disclosure** approach. Once a fix is available, we will publish a security advisory. We ask that you allow reasonable time for remediation before any public disclosure.
